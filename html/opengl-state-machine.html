OpenGL 为什么是一个状态机？

因为大多数 OpenGL 调用不是把全部参数一次性交给绘制函数，而是在持续修改“当前 OpenGL Context”中的状态。后续命令读取这些当前状态，决定操作哪个对象、怎样解释数据以及怎样产生像素。

01 · Definition

“状态机”到底是什么意思

它不是说 OpenGL 内部只有一个简单状态，也不是某种设计模式名称。它强调的是：OpenGL Context 保存大量当前值，命令会修改或消费这些值。

OpenGL 的行为不仅取决于你现在调用了哪个函数，还取决于在此之前留下了什么状态。
01
状态会持续存在

glBindTexture 绑定后不会在下一行自动失效，直到绑定另一张纹理或改变 Context。

02
许多命令操作“当前对象”

glBufferData 没有 VBO 参数，它修改当前绑定到目标上的 Buffer Object。

03
绘制命令消费一组状态

glDrawElements 会同时依赖当前 Program、VAO、EBO、纹理、Framebuffer 和各种渲染开关。

03 · State Families

Context 里保存了什么

OpenGL 状态很多，但学习阶段可以先分为三组：绑定到当前槽位的对象、固定功能渲染设置，以及绘制输出目标。

对象与绑定

  • Current Shader Program
  • Current VAO
  • Buffer bindings
  • 每个 Texture Unit 的纹理绑定
  • Framebuffer binding

渲染行为

  • Viewport 与 Scissor
  • Depth Test 与 Depth Function
  • Blend 与 Blend Function
  • Cull Face 与 Front Face
  • Clear Color / Depth / Stencil

对象内部状态

  • VAO 的顶点属性与 EBO
  • Texture 的过滤和环绕方式
  • Framebuffer 的附件
  • Program 的 uniform 值
  • Buffer 的数据存储
04 · Captured State

VAO 为什么能“记住”VBO

状态机最容易混淆的地方，是有些调用只修改全局当前绑定，有些调用还会把当时的绑定关系记录到另一个对象内部。

调用发生时

glBindVertexArray(vao1);
glBindBuffer(GL_ARRAY_BUFFER, vbo7);

glVertexAttribPointer(
    0, 3, GL_FLOAT, GL_FALSE,
    8 * sizeof(float), (void*)0
);

glVertexAttribPointer 会查看当前绑定的 VAO 和当前 `GL_ARRAY_BUFFER`，然后把属性格式以及 VBO 关联记录到 VAO。

→

VAO 1 内部记录

Attribute 0 BufferVBO 7
Components3 × float
Stride8 × float
Offset0 bytes
Element BufferEBO 9

之后即使 `GL_ARRAY_BUFFER` 绑定改变，重新绑定 VAO 1 时，attribute 0 仍知道从 VBO 7 读取。

05 · State Leakage

忘记恢复状态会怎样

状态不会因为一次绘制结束而恢复默认值。对象 A 开启的混合会继续影响对象 B，这种跨绘制的意外影响通常称为状态泄漏。

`5-texture/main.cpp` 中的初始化部分是在构建状态；渲染循环则是在为每次 Draw Call 选择正确状态并消费它们。

01glBindVertexArray(VAO)

02glBindBuffer(GL_ARRAY_BUFFER, VBO)

08 · Final Model

最后记住这三个动作

以后看到任何 OpenGL API，都可以先判断它属于哪一种动作，复杂代码就会变成可追踪的状态变化。

Bind · 选择当前对象例如 `glBindBuffer`、`glBindTexture`、`glBindVertexArray`。
Configure · 修改当前状态例如 `glBufferData`、`glTexParameteri`、`glEnable`、`glViewport`。
Consume · 使用当前状态例如 `glDrawArrays`、`glDrawElements`、`glClear`。
建议打开状态检查器，故意跳过 Program、VAO 或 Texture 绑定后点击 Draw，再回到 `5-texture/main.cpp` 对照每一行。
